# 贡献指南

感谢您对贡献 rate-limit 的兴趣！我们欢迎来自社区的贡献。

## 开始

1. **Fork 仓库**
   ```bash
   git clone https://github.com/vextjs/rate-limit.git
   cd rate-limit
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **运行测试**
   ```bash
   npm test
   ```

## 开发工作流

1. 为您的功能或错误修复创建新分支
   ```bash
   git checkout -b feature/my-feature
   ```

2. 进行修改

3. 运行测试和代码检查
   ```bash
   npm test
   npm run lint
   ```

4. 提交您的更改
   ```bash
   git commit -m "添加我的功能"
   ```

5. 推送到您的 fork
   ```bash
   git push origin feature/my-feature
   ```

6. 创建 Pull Request

## 代码风格

- 使用项目中提供的 ESLint 配置
- 遵循现有的代码约定
- 编写清晰、描述性的提交消息
- 为新功能添加测试
- 根据需要更新文档

## 测试

- 所有新功能必须包含测试
- 保持或提高测试覆盖率
- 运行 `npm run coverage` 检查覆盖率

## Pull Request 指南

- 保持 PR 专注于单一功能或错误修复
- 包含测试和文档
- 确保所有测试通过
- 如适用，更新 CHANGELOG.md
- 引用任何相关的问题

## 报告问题

- 使用 GitHub Issues 报告错误
- 包含清晰的重现步骤
- 提供环境详细信息（Node.js 版本、操作系统等）
- 包含相关的代码示例

## 行为准则

- 尊重和包容
- 欢迎新人
- 专注于建设性反馈

感谢您的贡献！

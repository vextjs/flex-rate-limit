## 报告漏洞

如果您在 rate-limit 中发现安全漏洞，请通过电子邮件报告：

**邮箱**: security@vextjs.com

请包含以下信息：

1. 漏洞描述
2. 重现步骤
3. 潜在影响
4. 建议修复（如有）

我们将在 48 小时内回复安全报告，并力争在 7 天内发布针对严重漏洞的修复。

## 安全最佳实践

在生产环境中使用 rate-limit 时：

1. **对分布式系统使用 Redis 存储**
   - 内存存储不会在实例间共享
   - Redis 提供持久化的分布式速率限制

2. **配置适当的限制**
   - 根据应用需求设置合理的限制
   - 考虑为不同端点设置不同的限制

3. **安全的键生成**
   - 使用安全标识符（用户 ID、API 密钥）
   - 注意基于 IP 的速率限制可能存在 IP 伪造问题

4. **监控速率限制命中情况**
   - 记录超过速率限制的事件
   - 对可疑模式发出警报

5. **保持依赖项更新**
   ```bash
   npm audit
   npm update
   ```

## 已知安全考虑事项

1. **分布式系统**：内存存储不会跨多个服务器同步
2. **竞态条件**：Redis 存储使用原子操作来防止竞态条件
3. **DoS 防护**：速率限制有助于防止 DoS 攻击，但不是完整的解决方案

## 披露政策

我们遵循负责任的披露实践，将会：

1. 确认收到漏洞报告
2. 提供修复的预计时间表
3. 在安全公告中注明报告者（如果需要）
4. 尽快发布安全补丁

感谢您帮助保持 rate-limit 的安全！
